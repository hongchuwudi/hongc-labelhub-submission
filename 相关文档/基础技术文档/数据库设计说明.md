# 数据库设计说明

## 设计原则

1. **禁用 ENUM**：所有状态/类型字段使用 VARCHAR，枚举定义在应用层 `state_machine/` 模块
2. **外键级联**：数据集→条目、任务→条目/结果/审核记录 使用 CASCADE DELETE
3. **SET NULL 保护**：ai_agents、label_schemas 等对 tasks 使用 SET NULL，保留历史记录
4. **版本快照**：Schema 发布时 schema_snapshot 存入 Task，保证异步任务读取一致性
5. **独立 AiReview 表**：AI 审核结果与标注结果分离，关注点分离，多轮重跑可追溯

## 12 张表结构

| 表名 | 用途 | 关键字段 |
|------|------|---------|
| `users` | 用户 | id, name, email(UK), role(owner/labeler/reviewer/ai_agent), password_hash |
| `datasets` | 数据集 | id, name, format(json/jsonl/csv/excel/xlsx), item_count, owner_id(FK) |
| `dataset_items` | 数据条目 | id, dataset_id(FK CASCADE), index, data(JSON) |
| `label_schemas` | 标注模板 | id, name, version, schema(JSON), version_history(JSON), owner_id(FK) |
| `tasks` | 任务 | id, title, dataset_id(FK), schema_id(FK), ai_agent_id(FK), owner_id(FK), status, quota, distribution_strategy |
| `task_items` | 任务条目 | id, task_id(FK CASCADE), dataset_item_id(FK), labeler_id(FK), status, flow_history(JSON), UK(task_id, dataset_item_id) |
| `results` | 标注结果 | id, task_id(FK CASCADE), item_id(FK), labeler_id, data(JSON), round, status(6态), comment, reviewer_id(FK), ai_scores(JSON) |
| `ai_reviews` | AI审核记录 | id, task_id(FK), result_id(FK UNIQUE), agent_id(FK), verdict, overall_score, dimensions(JSON), status, model, prompt_template, duration_ms |
| `ai_agents` | AI Agent | id, name, email(UK), system_prompt, scoring_dimensions(JSON), llm_model |
| `audit_logs` | 审计日志 | id, actor_id/name/role, entity_type/id, task_id(FK CASCADE), action, from_status→to_status, detail |
| `export_jobs` | 导出记录 | id, task_id(FK CASCADE), requested_by(FK), format, status, file_path, field_mapping, item_count |

## 核心关系链

```
users
  └── datasets ── dataset_items
  └── label_schemas
  └── ai_agents
  └── tasks ── task_items (via dataset_items)
          └── results ── ai_reviews (1:1)
          └── audit_logs
          └── export_jobs
```

## 关键索引

- `task_items(task_id, labeler_id, status)` — 标注工作台条目查询
- `ai_reviews(task_id)`, `ai_reviews(agent_id)` — AI 审核统计聚合
- `ai_reviews(result_id)` UNIQUE — 一个结果一个审核
