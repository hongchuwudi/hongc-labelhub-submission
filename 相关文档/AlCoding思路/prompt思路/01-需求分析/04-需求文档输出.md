# LabelHub — 需求规格说明书 (PRD)

## 一、项目概述

**一句话描述**：LabelHub 是一个覆盖「数据生产 → AI 预审 → 人工审核 → 多格式导出」全生命周期的 AI 数据标注平台。

**项目背景**：大模型与 Agent 训练极度依赖高质量业务数据，现有标注工具要么功能覆盖不全（如 Doccano 不支持可视化模板搭建），要么审核流程缺失（标注完成后无法人工多轮校验），要么 AI 辅助不够灵活。LabelHub 的目标是填补这些空白。

**目标用户**：
- 任务负责人（PM/运营/算法工程师）：负责数据生产的全流程管理
- 标注员（众包/行业专家）：领取任务、在线作答
- 审核员（质检/专家）：人工审核、质量把控

## 二、角色定义

| 角色 | 职责 | 能力边界 |
|------|------|---------|
| Owner | 数据全生命周期管理 | 数据集 CRUD、Schema 搭建、任务管理、AI 配置、导出、看板 |
| Labeler | 数据标注生产 | 任务广场、领取任务、在线作答、提交/修改打回 |
| Reviewer | 质量把控 | 审核列表、复审/终审、打回/通过、查看 AI 评分 |
| AI Agent | 自动预审 | 拉取提交、LLM 评测、写回评分 |

## 四、非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | Schema 50+ 字段表单不卡顿；LLM 调用不阻塞标注；导出大任务分批处理 |
| 安全 | 密码 bcrypt 哈希；JWT 双 Token；标签员数据隔离；角色守卫；防 SQL 注入（ORM） |
| 可维护性 | TypeScript strict（禁止 any）；状态机枚举在应用层；模块化目录结构 |
| 体验 | 错误提示友好；操作可逆（草稿恢复）；1280×800 + 1920×1080 表现良好 |

## 五、核心数据流

```
[Owner] 导入数据集(json/jsonl/csv/excel) → 创建 Schema(拖拽搭建) → 创建任务(绑定数据+Schema+AI)
→ [Owner] 发布任务 → task_items 批量生成
→ [Labeler] 浏览广场 → 领取任务 → Schema 驱动渲染表单 → 逐题作答 → 提交 Result
→ [MQ] 消息入队 → AI Agent 消费 → 调 LLM → 多维度打分 → 写 AiReview + 更新 Result 状态
→ [Reviewer] 审核队列 → 复审/终审 → 通过→入库 or 打回→Labeler 修改
→ [Owner] 导出（JSON/JSONL/CSV/Excel）
```

## 六、验收标准

| 核心功能 | 验收条件 |
|---------|---------|
| Schema 搭建器 | 拖拽 10 种物料搭建表单 → 导出 JSON Schema → 导入重建 → 布局一致 |
| Schema 渲染器 | 标注员工作台正确渲染搭建产物 → 字段校验生效 → 条件显示生效 |
| 任务发布 | 创建 → 绑定 → 发布 → 确认 task_items 生成 → 广场可见 |
| 标注作答 | 领取 → 逐题作答 → 草稿恢复 → 提交 → 状态变 submitted |
| AI 预审 | 提交 → agent 自动处理 → AiReview 写入 → result 状态更新 |
| 人工审核 | 复审通过 → 终审通过 → warehouse → 打回 → labeler 修改 → 重审 |
| 导出 | 创建导出 → 下载 JSON → 结构正确 → CSV/Excel 同内容 |
| 类型检查 | `npx tsc --noEmit` 零错误 |
| 测试 | `pytest -v` 全绿 |
