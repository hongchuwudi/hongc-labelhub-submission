# 角色与权限分析

## 我的提示词

```
你是一个权限架构师，请为 LabelHub 数据标注平台设计角色与权限模型。

平台有 4 种角色：
1. 任务负责人 (Owner) — 创建任务、搭建模板、配置审核、导出数据
2. 标注员 (Labeler) — 浏览任务广场、领取任务、在线作答、提交/修改
3. AI 审核 Agent (System) — 拉取提交数据、按规则打分、写回结果
4. 人工审核员 (Reviewer) — 多级审核（复审/终审）、打回/通过/修订

要求：
1. 列出每种角色能看到的前端页面和能执行的操作
2. 标注员之间是否应该互相看到对方的数据？为什么
3. Owner 和 Reviewer 的权限边界在哪里
4. AI Agent 作为一个"系统角色"需要哪些特殊权限
5. 设计前端路由守卫方案（登录守卫 + 角色守卫）

输出：角色权限矩阵表 + 路由守卫代码思路
```

---

## AI 的回答

### 1. 角色权限矩阵

| 页面/功能 | Owner | Reviewer | Labeler | AI Agent |
|-----------|-------|----------|---------|----------|
| 数据集管理（CRUD+导入） | ✅ | ❌ | ❌ | ❌ |
| Schema 设计器（搭建/版本） | ✅ | ❌ | ❌ | ❌ |
| 任务管理（CRUD+发布+暂停） | ✅ | ❌ | ❌ | ❌ |
| AI 配置（Agent CRUD） | ✅ | ❌ | ❌ | ❌ |
| AI 预审监控（查看/重跑） | ✅ | ✅ | ❌ | ❌ |
| 人工审核中心（复审/终审） | ✅ | ✅ | ❌ | ❌ |
| 数据看板 | ✅ | ❌ | ❌ | ❌ |
| 导出管理 | ✅ | ❌ | ❌ | ❌ |
| 任务广场（浏览/领取） | ❌ | ❌ | ✅ | ❌ |
| 标注工作台（作答/提交） | ❌ | ❌ | ✅ | ❌ |
| 我的贡献/收益 | ❌ | ❌ | ✅ | ❌ |
| 读取 LabelResult 数据 | ✅ | ✅ | 仅自己的 | ✅ |
| 修改 LabelResult 状态 | ✅ | ✅ | 仅自己 reject 的 | ✅ |
| 写 audit_log | ✅ | ✅ | ❌ | ✅ |

### 2. 标注员数据隔离

**结论：标注员不能看到其他标注员的数据。**

理由：
- 数据标注要求独立性，互相看到会引入 bias
- 防止抄袭/互相影响
- API 查询时按 labeler_id 强制过滤

### 3. Owner vs Reviewer 权限边界

- **Owner**：管理任务全生命周期（创建/发布/暂停/结束），配置 AI 规则，查看全量数据，导出
- **Reviewer**：只负责审核"已提交到审核队列"的数据，不能改任务配置、不能导出数据、不能创建/编辑 AI Agent
- Owner 可以兼任 Reviewer（拥有 reviewer 角色的全部能力）

### 4. AI Agent 系统角色设计

AI Agent 需要一个"系统用户在数据库中存在"（User 表中有记录，role=ai_agent）才能：
- 写入 audit_log 时记录 actor 信息（谁做了审核操作）
- 写入 LabelResult.ai_scores 时标记 labeler_type="ai"

特殊设计：
- AI Agent 也有 email + password_hash（密码随机生成，不对外暴露）
- AI Agent 不能通过 Web 登录（路由守卫拒绝 ai_agent 角色访问工作台）
- AI Agent 的 API 调用通过 AgentPool 内部发起，不经过认证中间件

### 5. 前端路由守卫方案

```tsx
// 登录守卫 — 未登录重定向 /login
function ProtectedRoute() {
  const { isLoggedIn, authReady } = useStore();
  if (!authReady) return <Spin />;
  if (!isLoggedIn) return <Navigate to="/login" />;
  return <Outlet />;
}

// 角色守卫 — 角色不匹配显示 403 或重定向
function RoleRoute({ roles }: { roles: string[] }) {
  const { user } = useStore();
  if (!roles.includes(user.role)) return <Result status="403" />;
  return <Outlet />;
}
```

路由嵌套结构：

```
AppLayout (公共)
├── /login
├── /register
└── /
ProtectedRoute ← 登录守卫
└── WorkspaceLayout (工作台)
    ├── RoleRoute roles=["owner"]
    │   ├── /datasets、/schemas、/tasks、/ai-configs、/dashboard、/export-history
    ├── RoleRoute roles=["owner","reviewer"]
    │   ├── /review、/ai-review
    └── RoleRoute roles=["labeler"]
        ├── /labeling、/earnings
```
