# 项目背景调研

## 我的提示词

```
你是一个全栈工程师，我正在计划做一个 AI 数据标注平台。请帮我做项目背景调研：

1. 分析当前标注平台市场上的主流产品（LabelStudio、Doccano、Labelbox 等），它们的核心功能是什么，有什么不足
2. 在大模型和 Agent 训练的场景下，数据标注平台需要具备哪些独特能力
3. 一个"人机协作"的标注平台应该包含哪些角色和流程
4. 给出一个项目应该具备的核心功能列表，按优先级排序

请用结构化的方式输出调研结论，每个要点给出分析理由。
```

---

## AI 的回答

### 1. 主流标注平台分析

| 产品 | 核心功能 | 不足 |
|------|---------|------|
| **LabelStudio** | 开源、多数据类型支持（文本/图像/音频/视频）、丰富的标注模板、API 完善 | 界面复杂学习成本高、中文支持弱、AI 辅助功能不够灵活、定制化成本高 |
| **Doccano** | 轻量级、文本序列标注/分类/情感分析、简单易用 | 功能单一只适合文本、无可视化模板搭建、无审核流转 |
| **Labelbox** | 商业产品、AI 辅助标注（模型预标注）、团队协作、多模态 | 闭源收费、国内部署困难、审核工作流不够灵活 |

**结论**：现有工具要么功能覆盖不全，要么审核流程缺失，要么 AI 辅助不够灵活。需要一个既能可视化搭建表单、又能配置 AI 审核标准、还能串联人工审核流转的贯穿式平台。

### 2. 大模型/Agent 场景的独特需求

- **动态表单搭建** — 不同微调任务需要不同标注字段（摘要/分类/评分/问题对等），不能写死表单，需要拖拽搭建
- **AI 预审能力** — 标注数据质量直接影响模型效果，需要用 LLM 做第一道自动质检，且评测标准要可配置
- **多轮审核流转** — SFT/RLHF 数据需要多轮人工校验，审核链路比传统标注长
- **结构化导出** — 导出数据要能直接喂给训练框架（JSONL/CSV），字段映射灵活

### 3. 人机协作的角色与流程

```
角色关系：
  Owner（任务负责人）→ 建任务、搭模板、配规则
  Labeler（标注员）→ 领取、作答、提交、修改
  AI Agent（系统）→ 自动预审、打分、分类
  Reviewer（审核员）→ 复审、终审、打回

流程：
  数据导入 → Owner搭建模板 → 创建任务 → Labeler领取标注 → AI自动预审 → Reviewer人工审核 → 入库导出
```

### 4. 核心功能列表（按优先级）

| 优先级 | 功能模块 | 说明 |
|--------|---------|------|
| P0 | 可视化表单搭建器 | 拖拽搭建、Tab/分组布局、JSON Schema 导出 — **核心竞争力** |
| P0 | Schema 驱动表单渲染 | 标注员工作台的渲染引擎，与搭建器复用同一套 Schema |
| P0 | 任务状态机 | 草稿/发布/暂停/结束 + 审核 6 态流转 |
| P1 | AI 自动预审 Agent | 可配置评分维度、LLM 评测、失败重试 |
| P1 | 人工审核流转 | 复审/终审、打回/通过、审计日志 |
| P1 | 数据集管理 | 导入/导出/预览（JSON/JSONL/CSV/Excel） |
| P2 | 数据看板 | 统计图表、任务进度 |
| P2 | 多格式导出 | 异步导出 + 字段映射 |
