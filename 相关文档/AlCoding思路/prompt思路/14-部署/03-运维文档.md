# 运维文档

## 提示词

```
请为 LabelHub 编写运维文档。

1. 日常维护：
   - 查看服务状态：docker-compose ps
   - 重启服务：docker-compose restart backend
   - 重启全部：docker-compose down && docker-compose up -d
   - 查看日志：docker-compose logs -f --tail=100 backend

2. 数据库维护：
   - 连接数据库：docker-compose exec db mysql -u root -p labelhub
   - 备份：docker-compose exec db mysqldump -u root -p labelhub > backup.sql
   - 恢复：docker-compose exec -T db mysql -u root -p labelhub < backup.sql
   - 清理旧数据：DELETE FROM audit_logs WHERE created_at < DATE_SUB(NOW(), INTERVAL 90 DAY);

3. Redis 维护：
   - 连接：docker-compose exec redis redis-cli
   - 查看锁：KEYS *lock*
   - 清缓存：FLUSHDB（谨慎使用）

4. RabbitMQ 维护：
   - 管理面板：http://<服务器IP>:15672（默认账号 guest/guest）
   - 查看队列消息堆积
   - 清空队列（谨慎）

5. 故障排查：
   - 后端启动失败：检查 .env 配置是否完整，数据库是否可连接
   - AI 审核不触发：检查 RabbitMQ 是否正常运行，LLM API Key 是否有效
   - 前端白屏：检查浏览器 Console 报错，确认 API proxy 配置正确
   - 内存占用高：调整 docker-compose 中的内存限制

6. 安全加固：
   - 修改所有默认密码（MySQL root、RabbitMQ guest）
   - 配置 JWT_SECRET 为强随机字符串
   - 数据库不暴露公网（仅容器内访问）
   - HTTPS 配置（推荐用 Let's Encrypt + Certbot）
   - 定期更新依赖（npm audit / pip-audit）

7. 监控建议：
   - 用 docker stats 查看资源使用
   - 关键指标：API 响应时间、AI Agent 队列长度、磁盘使用率

请输出完整的运维文档。
```
