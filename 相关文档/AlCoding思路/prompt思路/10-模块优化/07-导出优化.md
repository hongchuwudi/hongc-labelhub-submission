# 导出功能完善

## 提示词

```
请完善数据导出功能：

1. 异步导出完善：
   - 创建导出任务 → 发布消息到 labelhub.export 队列
   - export_worker.py 消费队列：
     a. 查询 task 下所有已入库 (warehouse) 的 result
     b. 根据 format 生成文件内容
     c. 按 field_mapping 筛选/重命名字段
     d. 上传文件到 OSS
     e. 更新 export_job 状态为 done + 写入 file_path
   - 前端轮询查看导出进度（或 WebSocket 推送）

2. 四格式支持完善：
   - JSON：输出为数组 [{item, result, ai_scores, review_info}]，pretty print
   - JSONL：每行一个 JSON 对象，便于流式消费
   - CSV：拍平嵌套 JSON，表头用点号路径（如 result.field1）
   - Excel (XLSX)：用 openpyxl 生成，第一个 sheet 为数据，第二个 sheet 为审核记录

3. 字段映射配置：
   - 前端导出配置界面：
     列出来源字段（原始数据字段 + 标注结果字段 + AI 评分字段 + 审核记录字段）
     勾选要导出的字段
     可重命名导出列名
     可排序
   - 提交时传入 field_mapping: [{source, target, include}]
   - 后端按 field_mapping 生成文件

4. 导出历史完善：
   - 列表：任务名、格式、状态、题目数、文件大小、创建时间
   - 已完成的可下载
   - 失败的显示错误信息
   - 删除旧导出记录

5. 大任务导出优化：
   - 超过 10000 条的分批生成（每次 1000 条）
   - 避免一次性加载全部数据到内存

请逐个实现：JSON → JSONL → CSV → Excel，每完成一种格式验证导出文件结构正确。
```

## AI 的回答

已实现：export_worker 消费 labelhub.export 队列。JSON 导出（数组 + pretty print）。JSONL 导出（每行一个 JSON）。CSV 导出（嵌套 JSON 拍平，点号路径表头）。Excel 导出（sheet1=数据 + sheet2=审核记录）。field_mapping [{source, target, include}] 支持字段筛选和重命名。大任务分批处理（每批 1000 条）。OSS 上传 + 签名 URL 下载。前端轮询导出进度。ExportHistory 页面展示状态标签 + 下载按钮 + 删除旧记录。
