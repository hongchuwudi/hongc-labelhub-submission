# 任务状态机完善

## 提示词

```
现在任务模块的基础功能已完成，请完善任务状态机和工作流：

1. 任务状态机完善：
   - 检查所有合法状态转移是否正确
   - 发布任务时：将 dataset items 批量导入 task_items，设置状态为 pending
   - 发布时校验：必须绑定 dataset 和 schema（缺一不可）
   - 暂停时：标注员仍可提交已领取的题目，但不能领新题
   - 结束时：拒绝新的提交，但已完成的不影响
   - 每种状态变更都写 audit_log

2. 任务分发策略实现：
   - 先到先得（默认）：labeler 主动 claim，直到配额满
   - 配额抢单：每人最多领取 grab_limit 道题
   - 并发领单用 Redis SETNX 锁防超发

3. 任务广场完善：
   - 只展示已发布的任务
   - 支持搜索（标题/描述）
   - 任务卡片展示：标题、描述、题数、奖励、剩余配额、截止时间
   - 我领取的任务高亮标注

4. 任务详情页完善：
   - 进度统计：已完成/总数、通过率
   - 关联信息：数据集预览、Schema 快照查看
   - 操作按钮根据状态机动态显示
   - 发布前抽屉确认（展示任务概要：分配方式、奖励、截止时间等）

5. 审计日志查询：
   - GET /api/tasks/:id/audit-logs — 任务下所有审计日志
   - 前端渲染审计时间线，筛选 action 类型

请逐个验证状态转移：创建→发布→暂停→恢复→结束，每条路径都走一遍。

## AI 的回答

已实现：发布校验（必须绑定 dataset + schema），发布时从 dataset items 批量创建 task_items。暂停状态允许已领取提交、禁止新领取。结束状态拒绝所有新提交。先到先得 + 配额抢单分发策略。任务广场搜索+筛选。任务详情进度统计。发布前确认抽屉。审计日志查询 API + 前端时间线渲染。
