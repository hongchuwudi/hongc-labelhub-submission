# 任务管理模块 Demo

## 提示词

```
请帮我实现任务管理模块的 MVP 版本。

后端部分：
1. 创建 LabelTask 模型：
   - id, title, description, dataset_id(FK), schema_id(FK), schema_snapshot(JSON)
   - ai_agent_id(FK), owner_id(FK)
   - deadline, quota, reward_per_item
   - distribution_strategy(先到先得/指派/配额抢单), grab_limit
   - status(draft/published/paused/ended)
   - total_items, completed_items
   - created_at, updated_at

2. 创建 TaskItem 模型：
   - id, task_id(FK), dataset_item_id(FK)
   - status(pending/labeled/skipped)
   - labeler_id(FK), flow_history(JSON)
   - unique(task_id, dataset_item_id)

3. 创建状态机 (state_machine/task.py)：
   - draft → published / ended
   - published → paused / ended
   - paused → published / ended
   - ended → (无，终态)
   - 用 validate_transition → transit 校验

4. 任务 API (api/v1/tasks/)：
   - GET /api/tasks — 列表（支持状态筛选、分页）
   - POST /api/tasks — 创建（状态初始=draft）
   - GET /api/tasks/:id — 详情（含 dataset 信息、schema 信息）
   - PATCH /api/tasks/:id — 编辑
   - DELETE /api/tasks/:id — 删除

5. 任务服务层 (services/tasks/service.py)：
   - 创建任务时：复制 schema_snapshot（冻结模板版本）
   - 发布任务时：将 dataset 的 items 批量导入 task_items
   - 状态变更前：调用状态机校验 → transit

前端部分：
1. 任务列表页 (pages/TaskManage/components/TaskList.tsx)：
   - 表格：标题、数据集、Schema、状态标签（颜色区分）、题目完成进度、创建时间
   - 状态筛选 Tab：全部/草稿/发布中/已暂停/已结束
   - 新建按钮 → Modal 表单

2. 任务详情/编辑页：
   - 基础信息编辑：标题、描述（富文本）、截止时间、配额、奖励
   - 关联选择：选数据集、选 Schema
   - 发布/暂停/结束 操作按钮（根据状态机显示可用操作）
   - 确认抽屉：发布前展示任务概要信息

3. API 客户端 (api/tasks.ts)：
   - 封装所有任务接口调用

请按后端模型→状态机→服务层→API→前端 的顺序实现。
```

## AI 的回答

```
LabelTask model with all fields, TaskItem model, state machine (draft→published→paused→ended) with validate_transition, task CRUD API, publish flow (snapshot schema + generate task_items), TaskList page with status tabs + create/delete, status operation buttons gated by state machine
```
