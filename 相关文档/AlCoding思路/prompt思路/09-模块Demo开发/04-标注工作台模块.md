# 标注员工作台模块 Demo

## 提示词

```
请帮我实现标注员工作台的 MVP 版本。

后端部分：
1. 任务广场 API：
   - GET /api/tasks?status=published（任务列表，供标注员浏览）
   - POST /api/tasks/:id/claim — 领取任务
   - GET /api/tasks/:id/my-stats — 我的任务统计

2. 标注作答 API：
   - GET /api/tasks/:id/items — 我的待标注题目列表
   - GET /api/tasks/:id/items/:itemId — 单题详情（含原始数据）
   - POST /api/tasks/:id/results — 提交某题的标注结果
   - GET /api/tasks/:id/my-results — 我的提交列表
   - POST /api/tasks/:id/items/:itemId/skip — 跳过某题

3. 领单并发控制：
   - 配额校验：已领取人数 >= quota 时拒绝
   - 用 Redis SETNX 分布式锁防超发

前端部分：
1. 任务广场页 (pages/Labeling/index.tsx 首次加载)：
   - 搜索框 + 筛选
   - 任务卡片网格：标题、描述预览、奖励、题目数、剩余配额、截止时间
   - 点击任务卡片 → 进入作答页

2. 作答页 (pages/Labeling/Workbench.tsx)：
   - 左侧题目队列 (ItemQueue)：
     - 题目列表（序号 + 状态图标：未做/已做/跳过）
     - 进度条 (已完成/总数)
     - 点击切换题目
   - 中间作答区 (WorkArea)：
     - 显示题目原始数据（ShowItem 渲染）
     - Schema 驱动的表单（根据任务绑定的 Schema 动态渲染）
     - 表单渲染组件 (SchemaForm.tsx)：
       - 读取 Schema → 动态渲染 Formily 表单
       - 字段条件显示（reactions）
       - 前端校验（必填/长度/正则）
   - 右侧元信息面板 (MetaPanel)：任务信息、题目进度

3. 底部操作栏 (BottomBar)：
   - 上一题 / 下一题 按钮
   - 保存草稿按钮（自动调用后端保存）
   - 提交按钮（校验 → 提交）

4. 草稿自动保存：
   - 每 30 秒或字段变更后自动保存
   - 页面关闭前提示未保存
   - 刷新后恢复草稿

5. LLM 辅助功能：
   - 创建 LLM 触发 API (api/v1/common/llm.py)：
     POST /api/llm-trigger
     接收：task_id, item_id, field_key
     调用 LLM 返回建议文本
   - 前端在 LLMInteraction 字段旁边提供"AI 建议"按钮

请按后端 API→前端页面 的顺序实现。先做基础的表单渲染，再逐步加草稿/LLM辅助等features。
```

## AI 的回答

```
task square API (list published tasks), claim API with Redis SETNX lock, items list/get API, submit result API, skip API, my-results/my-stats APIs, Labeling index page (task cards grid with search/filter), Workbench page (ItemQueue left + WorkArea center + MetaPanel right), SchemaForm rendering engine using Formily, BottomBar with prev/next/submit, auto-save draft with debounce, LLM trigger API and frontend button
```
