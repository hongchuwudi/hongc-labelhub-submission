# 导出与看板模块 Demo

## 提示词

```
请帮我实现数据导出和数据看板模块的 MVP 版本。

后端部分：
1. 创建 ExportJob 模型 (models/common/export.py)：
   - id, task_id, requested_by
   - format(json/jsonl/csv/xlsx), status(pending/processing/done/failed)
   - file_path, file_name, field_mapping(JSON)
   - error_message, item_count
   - created_at, finished_at

2. 导出 API：
   - POST /api/tasks/:id/exports — 创建导出任务
   - GET /api/tasks/:id/exports — 导出历史列表
   - GET /api/exports/:id/download — 下载导出文件

3. 导出服务 (services/common/export.py)：
   - create_export_job(task_id, format, field_mapping)
   - 入 MQ 队列 labelhub.export
   - export_worker 消费队列 → 根据 format 生成文件 → 上传 OSS → 更新 job 状态

4. 看板 API (api/v1/common/dashboard.py)：
   - GET /api/dashboard/stats — 概览统计
   - GET /api/dashboard/tasks-progress — 任务进度
   - GET /api/dashboard/datasets-stats — 数据集统计

前端部分：
1. 导出中心 (pages/TaskManage/ExportPage.tsx)：
   - 任务选择 → 格式选择（单选：JSON/JSONL/CSV/Excel）
   - 字段映射配置：选择导出哪些字段、是否包含审核记录
   - 创建导出 → 显示进度 → 下载

2. 导出历史 (pages/ExportHistory/index.tsx)：
   - 表格：任务名、格式、状态、题目数、创建时间、下载链接
   - 状态标签：处理中/完成/失败

3. 数据看板 (pages/Dashboard/index.tsx)：
   - 概览卡片：总任务数、总题目数、总标注数、通过率
   - 任务进度图（ECharts 柱状图）
   - 数据集统计（ECharts 饼图）
   - AI 审核通过率趋势

请按导出→看板的顺序实现。
```

## AI 的回答

```
ExportJob model (format/status/file_path/field_mapping/error_message), export create/list APIs, export service (MQ→worker→generate file→upload OSS→update status), dashboard stats/tasks-progress/datasets-stats APIs, ExportPage (task select + format select + field mapping), ExportHistory page (table with download links), Dashboard page (stats cards + ECharts bar/pie charts)
```
