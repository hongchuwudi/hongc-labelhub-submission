# 并发与边界测试

## 提示词

```
请对 LabelHub 进行并发和边界测试。

### 并发测试场景

1. 多个标注员同时领取同一任务：
   - 准备：一个任务配额=3，同时用 5 个 Labeler 账号发起领取
   - 验证：只有 3 人领取成功，另外 2 人收到"配额已满"
   - 验证：task_items 分配数量正确

2. 多个 Reviewer 同时审核同一题：
   - 准备：两个 Reviewer 同时点"通过"按钮
   - 验证：只有第一个成功，第二个收到"状态不允许此操作"

3. 标注员提交时管理员暂停任务：
   - 准备：Labeler 填写到一半时 Owner 暂停任务
   - 验证：提交时返回错误提示"任务已暂停"

### 边界测试场景

1. 空数据集创建任务 → 发布时提示"数据集为空"

2. 导入超大文件（50MB JSON）：
   - 验证：后端不崩溃
   - 验证：返回友好的错误提示"文件过大"

3. 极端数量的 Schema 字段（100+ 字段）：
   - 验证：Schema 设计器不卡顿
   - 验证：标注工作台表单渲染正常
   - 验证：导出文件结构完整

4. 任务截止时间在创建之前：
   - 验证：前端校验阻止

5. 特殊字符输入（SQL 注入、XSS 攻击）：
   - 在文本字段中输入 <script>alert(1)</script>
   - 验证：输出时被转义，不执行脚本
   - 验证：防 SQL 注入（ORM 参数化查询）

6. Token 过期处理：
   - 等待 access_token 过期
   - 验证：下一个请求自动 refresh
   - 验证：refresh_token 也过期时重定向登录页

7. 并发导出：
   - 同时对同一任务创建 3 个导出任务
   - 验证：每个导出独立执行，互不影响

每完成一个场景记录结果（通过/失败），失败的分析原因。
```
