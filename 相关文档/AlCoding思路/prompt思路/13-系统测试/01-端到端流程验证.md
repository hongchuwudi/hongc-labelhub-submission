# 端到端流程验证

## 提示词

```
请对 LabelHub 进行端到端全流程验证。

按照以下剧本逐一操作，验证每个环节功能正常：

### 剧本：完整数据生产流程

角色切换测试（先登出，再登入不同角色）：

**第一幕：Owner 创建任务**
1. 用 Owner 账号登录
2. 进入数据集管理 → 创建数据集"客服对话评测"
3. 导入 10 条题目数据（准备测试 JSON 文件）
4. 进入 Schema 设计器 → 拖拽搭建标注表单：
   - 包含字段：单行输入（摘要）、单选（情感标签：正面/负面/中性）、多行文本（详细评价）、LLM交互（AI辅助建议）
5. 保存 Schema 为"客服评测模板 v1"
6. 进入 AI 配置 → 创建 Agent"客服评测审核员"
   - 评分维度：相关性(40%)、准确性(30%)、完整性(30%)
7. 进入任务管理 → 创建任务"客服对话标注任务"
   - 选择数据集 + Schema + AI Agent
   - 设置配额 20、奖励 0.5/题
8. 发布任务

**第二幕：Labeler 领取并作答**
9. 退出 Owner → 用 Labeler 账号登录
10. 进入标注工作台 → 任务广场
11. 找到并领取"客服对话标注任务"
12. 逐题作答：
    - 验证 Schema 驱动的表单渲染正确
    - 验证字段校验（必填提示）
    - 点击 LLM 辅助按钮 → 查看 AI 建议
    - 填写完整 → 下一题（自动保存草稿）
13. 提交全部题目
14. 刷新页面 → 验证草稿恢复

**第三幕：AI 自动预审**
15. 提交后系统自动触发 AI 审核
16. 切换到 Owner 账号 → AI 预审监控页
17. 查看审核队列 → 确认有审核记录
18. 点击查看 AI 评分详情：
    - 维度评分
    - AI 评语
    - 通过的/打回的/标记人工复核的

**第四幕：Reviewer 人工审核**
19. 退出 Owner → 用 Reviewer 账号登录
20. 进入审核中心
21. 查看待审核队列
22. 点击某条 → 查看详情：
    - 原始数据展示
    - 标注结果展示
    - AI 评分面板
    - 审核时间线
23. 通过一条、打回一条（附理由）
24. 批量审核剩余

**第五幕：Labeler 处理打回**
25. 退出 Reviewer → 切换回 Labeler
26. 进入标注工作台 → 我的数据
27. 查看被达回题目 → 显示打回原因
28. 修改后重新提交

**第六幕：终审入库 + 导出**
29. 切换到 Reviewer → 对重新提交的题目做终审
30. 切换到 Owner → 数据看板 → 查看统计
31. 导出中心 → 导出 JSON + CSV → 下载验证文件

请记录每个环节是否正常，发现 Bug 立刻记录并在修复后重新验证该环节。
```
