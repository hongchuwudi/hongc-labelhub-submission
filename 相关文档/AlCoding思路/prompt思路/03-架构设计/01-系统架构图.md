# 系统架构图设计

## 我的提示词

```
请为 LabelHub 数据标注平台设计系统架构图。

系统组成：
- 前端：React SPA，三种角色对应不同页面
- 后端：FastAPI REST API，包含认证/数据集/Schema/任务/AI 审核/导出等模块
- AI Agent：线程池 + RabbitMQ 异步消费，调用 OpenAI 兼容 API
- 数据库：MySQL 存储业务数据，Redis 做缓存/锁/队列
- 消息队列：RabbitMQ 分发 AI 审核任务和导出任务
- 文件存储：OSS 存储上传文件和导出文件

请画出一张完整的架构图（用 ASCII art），包含：
1. 用户层 — 三种角色的浏览器入口
2. 前端层 — 路由守卫 + 各页面模块
3. API 网关层 — FastAPI 路由分组
4. 业务服务层 — Service 层各模块
5. 数据访问层 — SQLAlchemy ORM
6. 基础设施层 — MySQL + Redis + RabbitMQ + OSS
7. AI Agent 层 — AgentPool + MQ Consumer + LLM API

同时在每个层标注关键技术和数据流向。
```

数据流向：
  Owner 创建任务 → 发布 → Labeler 领取 → 提交 Result
  → MQ (labelhub.ai.review) → ai_worker 消费
  → AgentPool.submit() → Agent.run()
  → OpenAI API 调用 → 解析评分 JSON
  → 写入 ai_reviews + 更新 LabelResult
  → Reviewer 审核 → 入库 → 导出

这个架构的核心设计理念是 **分层解耦**：前端只管展示、API 只管路由、Service 只管业务逻辑、Agent 只管 LLM 调用，每层职责清晰。
