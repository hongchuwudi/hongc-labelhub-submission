# 状态机冒烟测试

## 提示词

```
请编写和运行状态机冒烟测试，确保所有状态转移正确。

测试文件：app/scripts/test_state_machine.py

验证内容：
1. 注册验证：
   - 所有 TRANSITION_TABLES 是否正确注册到全局表
   - 表名格式：LabelTask.status / TaskItem.status / LabelResult.status

2. 合法转移验证：
   - Task: draft→published→paused→published→ended（完整周期）
   - Task: draft→ended（直接终止）
   - TaskItem: pending→labeled / pending→skipped→labeled
   - LabelResult: submitted→ai_reviewing→review→final_review→warehouse
   - LabelResult: 任一阶段→rejected

3. 非法转移验证：
   - published→draft（不允许回退到草稿）
   - ended→published（已结束不能再发布）
   - warehouse→rejected（入库后不可打回）如果设计如此

4. 模型列类型检查：
   - 所有 status 列都是 VARCHAR（不是 ENUM）
   - 用 SHOW CREATE TABLE 或 inspect 验证

5. 实际流程验证：
   - 创建一条真实数据 → 在数据库里直接改 status → 确认状态机拒绝非法操作

6. 服务层集成：
   - 通过 Service 方法触发状态变更 → 确认 validate_transition 被正确调用

运行：python -m app.scripts.test_state_machine

每次修改状态机或 Service 代码后都必须跑一遍这个测试。

## AI 的回答

```
已创建 app/scripts/test_state_machine.py。验证了：所有 TRANSITION_TABLES 正确注册；Task/Item/Result 所有合法转移通过；所有非法转移被拒绝；所有 status 列为 VARCHAR（非 ENUM，经 SQLAlchemy inspect 确认）；实际数据流测试（创建记录→改状态→验证拒绝）；Service 层集成（status 变更通过 Service 正确触发 validate_transition）。运行 python -m app.scripts.test_state_machine —— 全部检查通过。
```
