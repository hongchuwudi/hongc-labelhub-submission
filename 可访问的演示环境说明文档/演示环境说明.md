## 三、演示流程

建议按以下顺序演示，覆盖三大角色完整链路：

```
1. [Owner]  登录 admin → 数据集管理 → Schema 设计器 → 创建任务 → 发布
2. [Labeler] 登录标注员 → 任务大厅认领 → 标注工作台作答 → 提交
3. [AI]     自动触发预审 → 在 AI 预审监控台查看评分
4. [Reviewer] 登录审核员 → 复审中心查看 → 通过/打回 → 批量操作
5. [Owner]  数据看板 → 导出中心 → 下载 JSON/CSV/Excel
```

### 5.1 进程清单

| 进程 | 管理方式 | 端口 | 内存 |
|------|---------|:---:|------|
| Nginx | Docker (`nginx:alpine`) | 80, 443 | ~10 MB |
| FastAPI (2 workers) | systemd `labelhub` | 8000 | ~260 MB |
| MySQL 8.0 | Docker (`mysql:8.0`) | 10010 | ~830 MB |
| Redis | Docker | 10001 | ~6 MB |
| RabbitMQ | Docker | 10030 | ~109 MB |

### 5.2 后端技术栈

| 组件 | 版本/说明 |
|------|----------|
| Python | 3.10.12 |
| FastAPI | 0.115+ |
| Gunicorn | 26.0.0（2 workers，Uvicorn worker） |
| SQLAlchemy | 2.0 |
| Pydantic | V2 |
| Agent 池 | ThreadPoolExecutor，7 并发 |

### 5.3 常用运维命令

```bash
# 查看后端服务状态
systemctl status labelhub

# 查看后端日志
journalctl -u labelhub -f

# 重启后端
systemctl restart labelhub

# 重载 Nginx
docker exec nginx-https nginx -s reload

# 查看 Nginx 日志
docker exec nginx-https cat /var/log/nginx/labelhub_error.log
```

## 六、部署脚本

项目提供一键部署脚本：

```bash
# 部署后端（更新代码 + 重启服务）
bash scripts/deploy-backend.sh

# 部署前端（构建 + 上传 + 刷新 CDN）
bash scripts/deploy-frontend.sh
```

## 七、注意事项

1. SSL 证书（Let's Encrypt）于 2026-09-07 到期，certbot 已配置自动续期
2. MySQL 容器内存限制为 1GB，Redis 和 RabbitMQ 共享剩余内存
3. 服务器总内存 3.3GB，当前使用约 2.1GB，余量约 1.2GB
4. 建议不要在服务器上同时运行大量压测（JMeter 等），以免 OOM
