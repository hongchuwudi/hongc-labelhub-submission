# LabelHub 状态机流转图

```mermaid
stateDiagram-v2
    direction LR

    %% ============================================================
    %% Level 1: Task 状态机
    %% ============================================================
    state "Task 状态机" as task_group {
        draft: 草稿
        published: 已发布
        paused: 已暂停
        ended_t: 已结束

        [*] → draft
        draft → published: Owner 发布
        draft → ended_t: Owner 结束（废弃）
        published → paused: Owner 暂停
        published → ended_t: Owner 结束
        paused → published: Owner 恢复
        paused → ended_t: Owner 结束
    }

    %% ============================================================
    %% Level 2: Item 状态机
    %% ============================================================
    state "Item 状态机" as item_group {
        pending: 待标注
        labeled: 已标注
        skipped: 已跳过

        [*] → pending
        pending → labeled: Labeler 提交标注
        pending → skipped: Labeler 跳过
        skipped → labeled: 回头标注
    }

    %% ============================================================
    %% Level 3: Result 状态机
    %% ============================================================
    state "Result 状态机（6态审核流转）" as result_group {
        submitted: 已提交
        ai_reviewing: AI 审核中
        review: 复审中
        final_review: 终审中
        warehouse: 已入库
        rejected: 已打回

        [*] → submitted
        submitted → ai_reviewing: Labeler 提交

        ai_reviewing → review: AI 通过/可疑
        ai_reviewing → rejected: AI 不合格打回

        review → final_review: 复审通过
        review → rejected: 复审打回

        final_review → warehouse: 终审通过→入库
        final_review → rejected: 终审打回

        rejected → review: 审核员重新打开\n（误操作恢复/申诉）

        warehouse → [*]
    }

    %% ============================================================
    %% 层级关系
    %% ============================================================
    published --> pending: Task 发布时\n批量创建 Item
    labeled --> submitted: Labeler 提交时\n创建 Result
```

### 角色图例

| 颜色 | 角色 | 操作 |
|------|------|------|
| ■ 紫色 | Owner | 发布/暂停/恢复/结束任务 |
| ■ 蓝色 | Labeler | 认领/标注/提交/跳过 |
| ■ 紫色 | AI Agent | 自动预审评分 |
| ■ 黄色 | Reviewer | 复审/终审通过或打回 |
| ■ 红色 | — | 打回通道 |

### 关键规则

- **非法转移拒绝** — 所有状态迁移通过 `validate_transition()` 校验，不在合法表中的转移自动报错
- **审计日志** — 每次状态迁移自动写入 `audit_logs`，记录 actor + from_status → to_status + detail
- **VARCHAR 存储** — 枚举值存储在应用层 `state_machine/` 模块，数据库不设 ENUM
