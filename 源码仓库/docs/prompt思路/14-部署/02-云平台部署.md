# 云平台部署

## 提示词

```
需要将 LabelHub 部署到腾讯云轻量应用服务器，对外提供 HTTPS 访问。
请给出详细的部署步骤。
```

### 部署步骤

#### 1. 环境准备

```bash
# SSH 登录服务器
ssh -i <key_path> root@49.232.16.204

# 安装 Docker
curl -fsSL https://get.docker.com | bash

# 启动中间件容器：MySQL / Redis / RabbitMQ
# （详见 01-Docker化.md 中间件容器章节）
docker ps
# 预期输出: mysql80-10010 / redis-10001 / redis-10000 / rabbitmq-10030
```

#### 2. 创建 Nginx HTTPS 容器

```bash
# 创建目录结构
mkdir -p /home/hongchu/nginx-https/{conf.d,certs,html}

# 先在 80 端口启动临时 Nginx（用于 certbot 验证）
docker run -d --name nginx-https \
  -p 80:80 -p 443:443 \
  -v /home/hongchu/nginx-https/conf.d:/etc/nginx/conf.d \
  -v /home/hongchu/nginx-https/certs:/etc/nginx/certs \
  -v /home/hongchu/labelhub-web:/home/hongchu/labelhub-web \
  nginx:alpine

# 申请 SSL 证书
certbot certonly --standalone -d labelhub.hongchu.xyz

# 将证书复制到 Nginx 挂载目录
cp /etc/letsencrypt/live/labelhub.hongchu.xyz/fullchain.pem /home/hongchu/nginx-https/certs/
cp /etc/letsencrypt/live/labelhub.hongchu.xyz/privkey.pem /home/hongchu/nginx-https/certs/private.pem

# 放入 Nginx 配置（详见 01-Docker化.md Nginx 配置章节）
# 重载 Nginx
docker exec nginx-https nginx -t && docker exec nginx-https nginx -s reload
```

#### 3. 部署后端

```bash
# 在本地机器执行
bash scripts/deploy-backend.sh

# 脚本自动完成：打包 → 上传 → 解压 → pip install → systemctl restart
# 验证后端
curl http://49.232.16.204:8000/docs
```

#### 4. 部署前端

```bash
# 在本地机器执行
bash scripts/deploy-frontend.sh

# 脚本自动完成：npm build → SCP 上传 → Nginx reload
# 验证前端
curl -I https://labelhub.hongchu.xyz
```

#### 5. 初始化种子数据

```bash
ssh root@49.232.16.204 "cd /opt/labelhub/backend && python3 -m app.scripts.seed"
```

#### 6. 验证部署

| 检查项 | 命令/地址 |
|--------|----------|
| 前端页面 | `https://labelhub.hongchu.xyz` |
| API 文档 | `http://49.232.16.204:8000/docs` |
| API 代理 | `curl https://labelhub.hongchu.xyz/api/auth/me` |
| 服务状态 | `ssh root@49.232.16.204 systemctl status labelhub` |
| 容器状态 | `ssh root@49.232.16.204 docker ps` |

### 部署脚本

项目提供两个一键部署脚本：

- `scripts/deploy-backend.sh` — 打包 → 上传 → 安装依赖 → systemd 重启
- `scripts/deploy-frontend.sh` — Vite 构建 → SCP 上传 → Nginx 配置更新 → reload
