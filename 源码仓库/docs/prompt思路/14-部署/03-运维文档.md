# 运维文档

## 提示词

```
LabelHub 已部署到生产环境，请编写日常运维文档。
```

---

## 实际运维方案

### 1. 日常维护

```bash
# 查看后端服务状态
ssh root@49.232.16.204 systemctl status labelhub

# 重启后端
ssh root@49.232.16.204 systemctl restart labelhub

# 查看后端日志（实时）
ssh root@49.232.16.204 journalctl -u labelhub -f

# 查看后端日志（最近 100 行）
ssh root@49.232.16.204 journalctl -u labelhub -n 100

# 查看所有容器状态
ssh root@49.232.16.204 docker ps
```

### 2. 版本更新

```bash
# 更新后端代码
bash scripts/deploy-backend.sh

# 更新前端代码
bash scripts/deploy-frontend.sh
```

### 3. 数据库维护

```bash
# 连接数据库（Docker 容器内）
ssh root@49.232.16.204 docker exec -it mysql80-10010 mysql -u root -p labelhub

# 备份数据库
ssh root@49.232.16.204 "docker exec mysql80-10010 mysqldump -u root -p<password> --single-transaction labelhub > /tmp/labelhub_backup_$(date +%Y%m%d).sql"

# 恢复数据库
ssh root@49.232.16.204 "docker exec -i mysql80-10010 mysql -u root -p<password> labelhub < /tmp/labelhub_backup.sql"
```

### 4. Redis 维护

```bash
# 连接 Redis（缓存实例）
ssh root@49.232.16.204 "docker exec -it redis-10001 redis-cli -a <password>"

# 查看当前锁 Key
> KEYS *lock*

# 查看 Refresh Token 白名单
> KEYS labelhub:refresh:*

# 清空指定 DB 缓存（DB 11）
> SELECT 11
> FLUSHDB
```

### 5. RabbitMQ 维护

管理面板：`http://49.232.16.204:15672`（默认账号 admin/<password>）

```bash
# 查看队列消息堆积
ssh root@49.232.16.204 "docker exec rabbitmq-10030 rabbitmqctl list_queues name messages"

# 查看未确认消息
ssh root@49.232.16.204 "docker exec rabbitmq-10030 rabbitmqctl list_queues name messages_unacknowledged"
```

### 6. 故障排查

| 症状 | 排查方向 |
|------|---------|
| 后端 502 | `systemctl status labelhub` 看服务是否 running；`journalctl -u labelhub -n 50` 查崩溃原因 |
| AI 审核不触发 | 检查 RabbitMQ 队列 `labelhub.ai.review` 是否有堆积；查 LLM API Key 是否过期 |
| 前端白屏 | 浏览器 F12 Console 看 JS 报错；确认 Nginx 配置 `/api/` proxy_pass 指向正确 |
| 导出失败 | 检查 OSS 配置是否正确；`docker logs rabbitmq-10030` 查看导出队列消费情况 |
| 磁盘满 | `df -h` 检查；清理 `journalctl --vacuum-size=500M`；清理 MySQL binlog |

### 7. SSL 证书续期

Let's Encrypt 证书有效期 90 天，需定期续期：

```bash
# 手动续期
ssh root@49.232.16.204 certbot renew

# 续期后拷贝到 Nginx 挂载目录
cp /etc/letsencrypt/live/labelhub.hongchu.xyz/fullchain.pem /home/hongchu/nginx-https/certs/
cp /etc/letsencrypt/live/labelhub.hongchu.xyz/privkey.pem /home/hongchu/nginx-https/certs/private.pem

# 重载 Nginx
docker exec nginx-https nginx -s reload

# 设置定时任务自动续期
# crontab -e
# 0 3 * * * certbot renew --quiet && cp ... && docker exec nginx-https nginx -s reload
```

### 8. 安全加固

| 措施 | 说明 |
|------|------|
| 数据库端口不暴露公网 | MySQL :10010 仅监听内网，防火墙不放行 |
| JWT_SECRET 随机化 | 生产环境使用随机 32 位字符串 |
| Nginx HTTPS 强制 | HTTP 80 端口自动 301 跳转 HTTPS |
| client_max_body_size | 限制上传 100MB，防止大文件攻击 |
| .env 不进 git | .env 在 .gitignore 中，仅通过 SCP 单独同步 |
