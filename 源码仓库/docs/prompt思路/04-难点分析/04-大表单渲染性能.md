# 大表单渲染性能 — 优化方案

## 1. Formily x-reactions 性能

**问题**：reactions 的依赖链过长时，更新一个字段可能触发连锁反应。

**方案**：限制依赖深度 + 用 `when` 代替 `fulfill`

```typescript
// 不好的写法 — 全量重算
{
  "x-reactions": {
    dependencies: [".fieldA", ".fieldB", ".fieldC"],
    fulfill: {
      state: {
        visible: "{{...复杂表达式...}}"
      }
    }
  }
}

// 好的写法 — 精确依赖 + 条件过滤
{
  "x-reactions": [
    {
      dependencies: [".fieldA"],
      when: "{{$deps[0] === 'show'}}",
      fulfill: { state: { visible: true } },
      otherwise: { state: { visible: false } }
    }
  ]
}
```

**技巧**：
- 每个 reaction 只依赖它真正需要的字段
- 用 `when` 做布尔判断，避免在 `fulfill` 中写复杂表达式
- 能拆成多个独立 reaction 的就不要写成一个大的

## 2. Tab 懒渲染

**结论：只渲染当前激活的 Tab**

```tsx
// TabbedForm.tsx — 根据 activeTab 选择性渲染
function TabbedForm({ schema, activeTab }: Props) {
  return (
    <div>
      {schema.tabs.map(tab => (
        <div 
          key={tab.id} 
          style={{ display: tab.id === activeTab ? 'block' : 'none' }}
        >
          <FormProvider form={form}>
            <TabContent tab={tab} fields={schema.fields} groups={schema.groups} />
          </FormProvider>
        </div>
      ))}
    </div>
  );
}
```

**为什么用 `display:none` 而不是条件渲染？**
- 条件渲染会导致 Tab 切换时销毁/重建 Formily 实例，丢失已填写的数据
- `display:none` 保留 DOM 和 Formily 状态，切换回来数据不丢
- 但未激活 Tab 的字段不参与 reactions 计算（Formily 自己处理）

## 3. 草稿自动保存

```typescript
// hooks/useAutoSave.ts

function useAutoSave(form: Form, taskId: number, itemId: number) {
  const saveRef = useRef<() => Promise<void>>();
  
  useEffect(() => {
    // 创建防抖保存函数
    const debouncedSave = debounce(async (values: any) => {
      try {
        setSaving(true);
        await saveDraftApi(taskId, itemId, values);
        setLastSaved(new Date());
      } finally {
        setSaving(false);
      }
    }, 3000);  // 3 秒防抖
    
    saveRef.current = debouncedSave;
    
    // 监听表单值变化
    const unsubscribe = form.subscribe(({ values }) => {
      saveRef.current?.(values);
    });
    
    return () => {
      unsubscribe();
      debouncedSave.cancel();
    };
  }, [form, taskId, itemId]);
  
  // 页面关闭前警告
  useEffect(() => {
    const handler = (e: BeforeUnloadEvent) => {
      if (form.modified) {
        e.preventDefault();
        e.returnValue = '';
      }
    };
    window.addEventListener('beforeunload', handler);
    return () => window.removeEventListener('beforeunload', handler);
  }, [form]);
}
```

**要点**：
- debounce 3 秒，避免每次按键都触发网络请求
- 记录 `lastSaved` 时间戳，给用户"已保存"的视觉反馈
- `beforeunload` 事件在表单有未保存修改时弹窗确认

## 4. 重量组件懒加载

```tsx
// 动态 import + React.lazy
const MonacoEditor = React.lazy(() => import('@monaco-editor/react'));
const ReactQuill = React.lazy(() => import('react-quill-new'));

// 字段渲染器中按需渲染
function FieldRenderer({ field }: { field: FieldConfig }) {
  switch (field.type) {
    case 'json':
      return (
        <Suspense fallback={<Skeleton.Input />}>
          <MonacoEditor height="200px" language="json" />
        </Suspense>
      );
    case 'richtext':
      return (
        <Suspense fallback={<Skeleton.Input />}>
          <ReactQuill theme="snow" />
        </Suspense>
      );
    default:
      return <Input />;
  }
}
```

## 5. 前端数据存储

```typescript
// store/labelingStore.ts — 标注工作台专用状态

interface LabelingState {
  // 任务信息
  task: TaskDetail | null;
  
  // 题目列表（只存 ID + 序号，不存完整数据）
  items: { id: number; index: number; status: string }[];
  currentIndex: number;
  
  // 当前题目的完整数据（只在需要时加载）
  currentItem: DatasetItem | null;
  
  // 草稿数据（按 itemId 索引）
  drafts: Record<number, Record<string, unknown>>;
  
  // 草稿元数据
  lastSaved: Record<number, Date>;
}
```

**策略**：
- 题目列表只存轻量信息（ID + 序号 + 状态），完整数据按需加载
- 上/下题切换时：保存当前草稿 → 加载下一题 → 恢复草稿
- 草稿数据存在 Zustand 中（内存），切换题目不丢

## 6. 题目切换优化

```tsx
// Workbench.tsx — 题目切换

function Workbench() {
  const [currentItemId, setCurrentItemId] = useState<number>();
  const [loading, setLoading] = useState(false);
  
  async function navigateToItem(itemId: number) {
    setLoading(true);
    
    // 1. 保存当前草稿（非阻塞）
    saveDraft(currentItemId, form.values).catch(console.warn);
    
    // 2. 加载新题目
    const item = await getItemDetail(taskId, itemId);
    
    // 3. 重置表单到新题目的值
    form.reset();  // 清空表单
    form.setValues(item.draft || {});  // 恢复草稿
    setCurrentItemId(itemId);
    
    setLoading(false);
  }
  
  return (
    <div>
      <ItemQueue 
        items={items} 
        currentId={currentItemId}
        onSelect={navigateToItem}
      />
      <WorkArea loading={loading}>
        <SchemaForm taskId={taskId} itemId={currentItemId} />
      </WorkArea>
      <BottomBar 
        onPrev={() => navigateToItem(prevItemId)}
        onNext={() => navigateToItem(nextItemId)}
      />
    </div>
  );
}
```

**优化要点**：
- 切换时先异步保存草稿（fire-and-forget），不阻塞加载
- 加载新题目显示 Skeleton，避免切换闪烁
- 预加载下一题的数据（鼠标悬停在下一题按钮上时触发预取）
