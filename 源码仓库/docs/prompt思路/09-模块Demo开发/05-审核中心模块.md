# 人工审核中心模块 Demo

## 提示词

```
请帮我实现人工审核中心的 MVP 版本。

后端部分：
1. 审核 API（在 tasks router 下）：
   - GET /api/tasks/:id/results — 提交结果列表（支持状态筛选）
   - GET /api/tasks/:id/results/:resultId — 提交详情（含原始数据 + AI评分）
   - PATCH /api/tasks/:id/results/:resultId — 审核操作（通过/打回）
   - POST /api/tasks/:id/results/batch-review — 批量审核

2. 审核状态机流转 (state_machine/result.py)：
   - submitted → ai_reviewing → review → final_review → warehouse
   - 任一阶段可 → rejected

3. 审核服务层：
   - review_result(result_id, action, comment)：
     - 校验 Reviewer 权限
     - 状态机校验当前状态是否允许审核
     - 执行 transit
     - 写入 audit_log
   - batch_review(result_ids, action, comment)

前端部分：
1. 审核列表页 (pages/Review/components/ReviewList.tsx)：
   - 顶部：任务选择器（下拉选任务）+ 状态筛选
   - 表格：题目标题、标注员、提交时间、AI评分、当前状态
   - 每条可点击进入详情
   - 批量选择 + 批量通过/打回

2. 审核详情页 (pages/Review/components/ReviewDetail.tsx)：
   - 左侧：原始数据展示（ShowItem）
   - 中间：标注结果展示（Schema 表单只读模式）
   - 右侧：AI 评分面板 (AiScorePanel)
     - 多维度评分条形图
     - AI 评语
     - 原始 Prompt 可展开查看
   - 底部：操作栏
     - 通过按钮（状态机允许时显示）
     - 打回按钮 → 弹出理由输入框
     - 审核意见输入框

3. 审核时间线 (components/AuditTimeline.tsx)：
   - 从 audit_logs 读取并渲染时间线
   - 显示：谁、什么时间、做了什么操作、从什么状态到什么状态

4. 批量审核：
   - 列表多选 → 弹出确认 Modal → 查看批量预览 → 确认 → 调用批量 API

请先做审核列表+详情，再做批量操作和时间线。
```

## AI 的回答

```
results list/detail/batch-review APIs, state machine 6-state flow, review service with status validation, ReviewList page (task selector + status filter + table with batch select), ReviewDetail page (raw data + annotation result + AI score panel + audit timeline), batch review with preview modal
```
