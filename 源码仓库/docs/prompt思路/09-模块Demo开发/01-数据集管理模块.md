# 数据集管理模块 Demo

## 提示词

```
请帮我实现数据集管理模块的 MVP（最小可用版本）。

后端部分：
1. 创建 Dataset 模型：
   - id, name, description, format(json/jsonl/csv/excel), item_count, owner_id, created_at

2. 创建 DatasetItem 模型：
   - id, dataset_id(FK), index, data(JSON), created_at

3. 数据集 API (api/v1/datasets/)：
   - GET /api/datasets — 列表（分页，按 owner 过滤）
   - POST /api/datasets — 创建
   - GET /api/datasets/:id — 详情
   - PATCH /api/datasets/:id — 编辑
   - DELETE /api/datasets/:id — 删除
   - GET /api/datasets/:id/items — 题目列表（分页）
   - POST /api/datasets/:id/items/batch — 批量导入题目（JSON 数组）
   - PATCH /api/datasets/:id/items/:itemId — 编辑单条题目
   - DELETE /api/datasets/:id/items/:itemId — 删除单条题目
   - GET /api/datasets/:id/items/export — 导出题目为 JSON/JSONL/CSV

4. 数据集服务层 (services/datasets/)：
   - 创建时自动设置 owner_id = 当前用户
   - 查询时自动按 owner_id 过滤（普通用户只能看自己创建的）
   - 级联删除：删除 dataset 时同时删除所有 items

前端部分：
1. 数据集列表页 (pages/Datasets/DatasetList.tsx)：
   - 表格展示：名称、格式、题目数、创建时间
   - 搜索 + 刷新
   - 新建按钮 → 弹出 Modal 表单（名称、描述、格式）

2. 数据集详情页 (pages/Datasets/DatasetDetail.tsx)：
   - 项目信息卡片
   - 题目列表（分页表格，展示 index + data 预览）
   - 批量导入按钮 → Modal 选择文件上传
   - 单条编辑选项
   - 删除选项（确认弹窗）

3. 数据导入页 (pages/Datasets/DataImport.tsx)：
   - 文件上传（支持 JSON/JSONL/CSV/Excel）
   - 预览解析后的数据
   - 确认导入

4. API 客户端 (api/datasets.ts)：
   - 封装所有数据集相关接口调用
   - 正确使用 TypeScript 类型（从 types/models/dataset.ts 导入）

请按后端模型→后端API→前端API→前端页面 的顺序逐步实现。每步完成先验证。
```

## AI 的回答

```
Dataset model (id/name/description/format/item_count/owner_id), DatasetItem model (id/dataset_id/index/data JSON), CRUD API endpoints, service layer with owner filtering + cascade delete, frontend DatasetList page (table + search + create modal), DatasetDetail page (item list + batch import + inline edit), DataImport page (file upload + preview)
```
