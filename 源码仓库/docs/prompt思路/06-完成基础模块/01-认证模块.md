### 后端实现

**User 模型** (`models/auth/user.py`)

```python
from sqlalchemy import Column, Integer, String, DateTime
from app.models.base import BaseModel

class User(BaseModel):
    __tablename__ = "user"

    id = Column(Integer, primary_key=True, autoincrement=True)
    name = Column(String(64), nullable=False)
    email = Column(String(128), unique=True, nullable=False)
    password_hash = Column(String(256), nullable=False)
    role = Column(String(32), nullable=False, default="labeler")  # VARCHAR, 非 ENUM
    avatar = Column(String(256), nullable=True)
    created_at = Column(DateTime, nullable=False, server_default=func.now())
```

**安全模块** (`infra/security.py`)

- `hash_password(password: str) -> str` — bcrypt 哈希
- `verify_password(plain: str, hashed: str) -> bool` — bcrypt 校验
- `create_access_token(user_id: int, role: str) -> str` — payload: `{sub, role, type: "access"}`, 有效期 30 分钟
- `create_refresh_token(user_id: int) -> str` — payload: `{sub, jti, type: "refresh"}`, 有效期 7 天，jti 写入 Redis SET 做白名单
- `get_current_user(token: str = Depends(oauth2_scheme))` — 从 Authorization Bearer 解码 access_token，查询数据库返回 User 对象
- `require_role(*roles: str)` — 依赖工厂，校验 `current_user.role` 是否在允许列表中

**JWT 双 Token 流**

- access_token 无状态验证，不依赖 Redis
- refresh_token 的 jti 在签发时写入 Redis SET；刷新时校验 jti 存在，校验通过后删除（一次性消费），生成新的 access_token + refresh_token
- 退出登录时删除 Redis 中对应 jti，实现 token 主动失效

**认证路由** (`api/v1/auth/router.py`)

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/register` | name + email + password，默认 role=labeler，bcrypt 哈希存储，返回 user + tokens |
| POST | `/api/auth/login` | email + password 校验，返回 `{access_token, refresh_token, user}` |
| POST | `/api/auth/refresh` | 接收 refresh_token，校验 jti 并一次性消费，返回新 token 对 |
| GET | `/api/auth/me` | 需登录，返回当前用户信息 |

### 前端实现

**API 客户端** (`api/client.ts`)

```typescript
const http = axios.create({ baseURL: "/api", timeout: 30000 });

// 请求拦截器：自动附加 Authorization Bearer token
http.interceptors.request.use((config) => {
  const token = useAuthStore.getState().accessToken;
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// 响应拦截器：401 自动刷新，其他请求排队等待
let isRefreshing = false;
let refreshQueue: (() => void)[] = [];

http.interceptors.response.use(null, async (error) => {
  if (error.response?.status === 401 && !error.config._retry) {
    if (isRefreshing) {
      return new Promise(resolve => refreshQueue.push(() => resolve(http(error.config))));
    }
    isRefreshing = true;
    try {
      const newTokens = await refreshTokenApi();
      // 重放排队中的请求
      refreshQueue.forEach(cb => cb());
      refreshQueue = [];
      return http(error.config);
    } catch {
      useAuthStore.getState().logout();
      window.location.href = "/login";
    } finally {
      isRefreshing = false;
    }
  }
  return Promise.reject(error);
});
```
