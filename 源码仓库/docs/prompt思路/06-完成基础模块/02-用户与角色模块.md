# 用户与角色管理模块

## 提示词

```
请实现 LabelHub 的注册和公共模块。

注册模块：
1. 后端注册 API：POST /api/auth/register
   - 接收 name, email, password
   - 验证邮箱唯一性
   - 密码 bcrypt 哈希后存储
   - 默认角色：labeler（前台注册）/ owner（通过 seed 脚本创建）
   - 返回用户信息 + token

2. 后端用户列表 API：GET /api/users
   - 按 role 筛选
   - 分页支持
   - 只返回同 role 的用户（或管理员可看全部）

3. 前端注册页面：
   - 表单：姓名 + 邮箱 + 密码 + 确认密码
   - 前端校验：邮箱格式、密码长度 >= 8、两次密码一致
   - 注册成功后自动登录并跳转到对应角色首页

4. 种子数据脚本 (scripts/seed.py)：
   - 创建默认 Owner 账号（从环境变量读取或使用固定值）
   - 创建几个 Labeler 测试账号
   - 创建 Reviewer 测试账号
   - 创建默认 AI Agent 账号

请实现并验证：注册→登录→看到的页面根据角色不同
```

### 后端实现

**注册 API** (`POST /api/auth/register`)

```python
@router.post("/register", response_model=AuthResponse)
async def register(data: RegisterRequest, db: Session = Depends(get_db)):
    # 校验邮箱唯一性
    existing = db.query(User).filter(User.email == data.email).first()
    if existing:
        raise ConflictException("email already registered")
    
    # 密码 bcrypt 哈希
    user = User(
        name=data.name,
        email=data.email,
        password_hash=hash_password(data.password),
        role="labeler"  # 前台注册默认角色固定为 labeler
    )
    db.add(user)
    db.commit()
    
    # 注册成功自动返回 token，方便前端自动登录
    access_token = create_access_token(user.id, user.role)
    refresh_token = create_refresh_token(user.id)
    return AuthResponse(user=user, access_token=access_token, refresh_token=refresh_token)
```

**用户列表 API** (`GET /api/users`)

- 查询参数：`role`（可选筛选）、`page`、`page_size`
- 按 role 过滤用户列表，支持分页
- 权限：owner 可查所有角色用户；reviewer/labeler 只可查同角色用户
