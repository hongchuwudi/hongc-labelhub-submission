# LabelHub 多级状态机设计

## 1. 任务状态机 (Task)

```
      ┌──────────┐
      │  draft   │  草稿（可编辑、可删除）
      └────┬─────┘
           │ publish / end
    ┌──────┴──────┐
    ▼             ▼
┌──────────┐  ┌──────────┐
│published │  │  ended   │  已结束（终态）
│  发布中   │  └──────────┘
└────┬─────┘
     │ pause
     ▼
┌──────────┐
│  paused  │  已暂停（不可领新题）
│  已暂停   │
└────┬─────┘
     │ resume(publish) / end
     │
     ├──→ published
     └──→ ended
```

**合法转移：**
| 从 | 到 | 说明 |
|----|-----|------|
| draft | published | 发布任务 |
| draft | ended | 直接终止 |
| published | paused | 暂停 |
| published | ended | 结束 |
| paused | published | 恢复 |
| paused | ended | 结束 |
| ended | — | 终态，不可转移 |

**非法转移示例：**
- published → draft（不允许回退到草稿）
- ended → published（已经结束不能再发布）
- paused → draft

## 2. 题目状态机 (TaskItem)

```
┌──────────┐
│ pending  │  待标注
└────┬─────┘
     │
  ┌──┴───┐
  ▼      ▼
┌────┐ ┌──────┐
│labeled│ │skipped│

└────┘ └──┬───┘
     ▲     │
     └─────┘  skipped 可重新标注 → labeled
```

## 3. 审核状态机 (LabelResult) — 关键

```
                          ┌──────────────────────────┐
                          │                          │
                          ▼                          │
┌───────────┐   ┌──────────────┐   ┌─────────┐   ┌──────────────┐
│ submitted │→ │ ai_reviewing │→ │ review  │→ │ final_review │
│  已提交    │   │   AI 审核中   │   │  复审    │   │    终审       │
└───────────┘   └──────┬───────┘   └────┬────┘   └──────┬───────┘
      │                │                │                │
      │         ┌──────┘         ┌──────┘         ┌──────┘
      ▼         ▼                ▼                ▼
┌─────────────────────────────────────────────────────┐
│                    rejected                          │
│                      打回                             │
└─────────────────────────────────────────────────────┘
                                          │
                              ┌───────────┘
                              ▼
                      ┌──────────────┐
                      │  warehouse   │  归档入库（终态）
                      └──────────────┘
```

**合法转移表：**

| 从 | 到 | 说明 |
|----|-----|------|
| submitted | ai_reviewing | 进入 AI 审核队列 |
| ai_reviewing | review | AI 通过 → 人工复审 |
| ai_reviewing | rejected | AI 直接打回 |
| review | final_review | 复审通过 → 终审 |
| review | rejected | 复审打回 |
| final_review | warehouse | 终审通过 → 入库 |
| final_review | rejected | 终审打回 |
| submitted/ai_reviewing/review/final_review | rejected | 任一阶段均可打回 |

**非法转移：**
- warehouse → 任何状态（入库后不可更改）
- ai_reviewing → warehouse（不能跳过人工审核）
- submitted → warehouse（必须经过审核流程）

## 4. 通用状态机框架

```python
# state_machine/base.py

# 全局注册表
TRANSITION_TABLES: Dict[str, Dict[str, Set[str]]] = {}

def register(name: str, table: dict):
    """注册状态转移表"""
    TRANSITION_TABLES[name] = table

def validate_transition(entity_type: str, from_status: str, to_status: str) -> bool:
    """校验状态转移是否合法"""
    table = TRANSITION_TABLES.get(entity_type)
    if not table:
        raise ValueError(f"Unknown entity type: {entity_type}")
    allowed = table.get(from_status, set())
    return to_status in allowed

def transit(entity, field: str, entity_type: str, 
            to_status: str, db: Session, actor_info: dict):
    """执行状态转移（校验 + 设置 + 审计）"""
    from_status = getattr(entity, field)
    
    if not validate_transition(entity_type, from_status, to_status):
        raise BadRequestException(
            f"非法状态转移: {entity_type} {from_status} → {to_status}"
        )
    
    # 设置新状态
    setattr(entity, field, to_status)
    
    # 写入审计日志
    audit_log = AuditLog(
        actor_id=actor_info["id"],
        actor_name=actor_info["name"], 
        actor_role=actor_info["role"],
        entity_type=entity_type,
        entity_id=entity.id,
        action=f"status_change",
        from_status=from_status,
        to_status=to_status,
        detail=json.dumps({"field": field})
    )
    db.add(audit_log)
    db.flush()
```

```python
# state_machine/task.py
class TaskStatus(str, Enum):
    DRAFT = "draft"
    PUBLISHED = "published"
    PAUSED = "paused"
    ENDED = "ended"

TASK_TRANSITIONS = {
    TaskStatus.DRAFT:     {TaskStatus.PUBLISHED, TaskStatus.ENDED},
    TaskStatus.PUBLISHED: {TaskStatus.PAUSED, TaskStatus.ENDED},
    TaskStatus.PAUSED:    {TaskStatus.PUBLISHED, TaskStatus.ENDED},
    TaskStatus.ENDED:     set(),
}
register("LabelTask.status", TASK_TRANSITIONS)

# state_machine/result.py
class ResultStatus(str, Enum):
    SUBMITTED = "submitted"
    AI_REVIEWING = "ai_reviewing"
    REJECTED = "rejected"
    REVIEW = "review"
    FINAL_REVIEW = "final_review"
    WAREHOUSE = "warehouse"

RESULT_TRANSITIONS = {
    ResultStatus.SUBMITTED:    {ResultStatus.AI_REVIEWING, ResultStatus.REJECTED},
    ResultStatus.AI_REVIEWING: {ResultStatus.REVIEW, ResultStatus.REJECTED},
    ResultStatus.REVIEW:       {ResultStatus.FINAL_REVIEW, ResultStatus.REJECTED},
    ResultStatus.FINAL_REVIEW: {ResultStatus.WAREHOUSE, ResultStatus.REJECTED},
    ResultStatus.REJECTED:     {ResultStatus.SUBMITTED},  # 修改后重新提交
    ResultStatus.WAREHOUSE:    set(),
}
register("LabelResult.status", RESULT_TRANSITIONS)
```

## 5. 并发控制：两个 Reviewer 同时审核同一题

**问题**：Reviewer A 和 Reviewer B 同时点"通过"，都读到 status=review，都认为可以转 final_review。

**方案：数据库乐观锁**

不额外加列，利用状态机本身做 CAS（比较并交换）：

```python
def review_result(db, result_id, action, reviewer_id, comment):
    # 在同一个事务中：查询 → 校验状态 → 更新
    result = db.query(LabelResult).filter_by(id=result_id).first()
    
    # 状态机校验（拿当前状态 check 转目标状态是否合法）
    if action == "approve":
        # review → final_review 合法，final_review → warehouse 合法
        new_status = get_next_status(result.status)
    elif action == "reject":
        new_status = ResultStatus.REJECTED
    
    # 如果中间有人改了状态，这里会失败
    if not validate_transition("LabelResult.status", result.status, new_status):
        raise ConflictException("状态已变更，请刷新后重试")
    
    transit(result, "status", "LabelResult.status", new_status, db, actor_info)
```

为什么不用 Redis 锁？因为审核是低频操作（人工操作，不是高并发），用状态机自带的校验就够。如果真的需要，可以加 DB 乐观锁（version 字段）。

```python
# app/scripts/test_state_machine.py
def test_no_enum_columns():
    """验证数据库中没有 ENUM 列"""
    inspector = inspect(engine)
    for table_name in inspector.get_table_names():
        for column in inspector.get_columns(table_name):
            col_type = str(column['type']).lower()
            assert 'enum' not in col_type, \
                f"表 {table_name}.{column['name']} 使用了 ENUM，请改为 VARCHAR"
```
