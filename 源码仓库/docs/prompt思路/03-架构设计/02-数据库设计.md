# 数据库设计

## 我的提示词

```
你是一个数据库架构师，请为 LabelHub 数据标注平台设计完整的数据库模型。

需求背景：
- 4 种角色：Owner/Labeler/Reviewer/AI Agent
- Owner 上传数据集（含多条题目），创建 Schema（表单模板），绑定到 Task 发布
- Labeler 领取 Task → 按 Schema 渲染表单 → 逐题作答 → 提交 Result
- AI Agent 拉取已提交 Result → 多维度打分 → 写入 AiReview
- Reviewer 查看 AiReview + Result → 复审/终审/打回
- 所有状态变更记录到 audit_logs
- 支持多格式异步导出（export_jobs）

请设计表结构，要求：
1. 列出所有表（约 10 张），每张表列出所有字段、类型、说明、索引
2. 标注主键、外键、唯一约束、级联删除规则
3. 状态/类型字段一律用 VARCHAR（禁止 ENUM），枚举值在应用层管理
4. 多轮审核结果（AiReview）和标注结果（LabelResult）要分开存，理由是什么
5. 版本管理的字段设计（Schema 有 version + 历史快照）
6. 审计日志表（audit_logs）的通用设计：actor/entity/action/from/to/detail

输出：完整的 SQL DDL 或者 Markdown 表结构文档
```

---

## AI 的回答

# LabelHub 数据库设计

## 设计原则

1. **禁止 ENUM** — 所有状态/类型/角色字段用 VARCHAR，枚举定义在应用层（state_machine/*.py）
2. **状态外键关联** — AiReview 独立成表，与 LabelResult 通过 result_id 关联（关注点分离）
3. **版本快照** — Schema 发布后冻结为 schema_snapshot 存入 Task，保证异步任务读取一致性
4. **审计全记录** — 所有状态变更自动写 audit_logs

### 8. ai_reviews — AI 审核记录表（独立存储）

**为什么独立成表？**
1. 关注点分离 — 人工标注结果 vs AI 审核结果是不同的事
2. 可追溯 — 每次审核独立一行，记录多轮重跑历史
3. 统计方便 — 对 AiReview 直接聚合查询（通过率、平均耗时），不需要解析 JSON

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | INT PK | 主键 | |
| task_id | INT FK | 任务 | INDEX |
| item_id | INT FK | 题目 | INDEX |
| result_id | INT FK→results.id | 审核对象 | UNIQUE |
| agent_id | INT FK→ai_agents.id | 审核 Agent | |
| verdict | VARCHAR(20) | pass/reject/human_review/error | INDEX |
| summary | TEXT | AI 总评 | |
| dimensions | JSON | [{name,score,reason}] | |
| overall_score | FLOAT | 综合分 0-100 | |
| model | VARCHAR(50) | 使用的 LLM 模型 | |
| prompt_template | TEXT | Prompt 模板 | 审计追溯 |
| prompt_vars | JSON | 模板变量值 | 审计追溯 |
| status | VARCHAR(20) | pending/processing/done/failed | |

## 关键外键删除策略

| 父表 | 子表 | 策略 | 说明 |
|------|------|------|------|
| users | datasets | CASCADE | 删用户→删数据集 |
| datasets | dataset_items | CASCADE | 删数据集→删题目 |
| tasks | task_items | CASCADE | 删任务→删关联 |
| tasks | results | CASCADE | 删任务→删结果 |
| tasks | ai_reviews | CASCADE | 删任务→删AI审核记录 |

注意：ai_agents、label_schemas、datasets 对 tasks 用 SET NULL（保留任务记录，关联置空）。
