# 后端单元测试

## 提示词

```
请为 LabelHub 后端编写单元测试。

测试框架：pytest + pytest-asyncio + httpx (TestClient)

1. 测试基础设施 (tests/conftest.py)：
   - 创建测试数据库（可用 SQLite :memory: 替代 MySQL）
   - 创建测试用 FastAPI TestClient
   - fixture: db_session（每个测试独立的数据库会话）
   - fixture: client（带认证 header 的 TestClient）

2. 认证模块测试 (tests/auth/)：
   - test_register：正常注册 → 返回用户 + token
   - test_register_duplicate_email：重复邮箱注册 → 409
   - test_login：正确密码登录 → 返回双 token
   - test_login_wrong_password：错误密码 → 401
   - test_refresh：用 refresh_token 换新 access_token
   - test_refresh_reuse：同一个 refresh_token 使用两次 → 第二次 401
   - test_me：带 Authorization 的请求获取当前用户

3. 数据集测试 (tests/datasets/)：
   - test_create_dataset：创建数据集
   - test_list_datasets：列表查询（验证分页）
   - test_get_dataset：获取详情
   - test_update_dataset：更新
   - test_delete_dataset：删除（级联验证 items 也被删除）
   - test_import_items：批量导入题目
   - test_owner_isolation：用户 A 看不到用户 B 的数据集

4. 任务测试 (tests/tasks/)：
   - test_create_task：创建任务（draft 状态）
   - test_publish_task：发布任务（验证 task_items 生成）
   - test_pause_task：暂停
   - test_end_task：结束
   - test_illegal_transition：非法状态转移 → 被拒绝
   - test_claim_task：领取任务
   - test_submit_result：提交标注结果
   - test_review_result：审核通过/打回

5. 状态机测试 (tests/common/test_state_machine.py)：
   - 验证所有合法转移
   - 验证所有非法转移被拒绝
   - 验证 audit_log 自动写入

每写完一个测试文件就运行 pytest -v 确保全绿。
```
