# 前端类型检查

## 提示词

```
请对前端项目执行完整的类型检查：

1. 运行类型检查命令：
   npx tsc -p tsconfig.app.json --noEmit

2. 解读报错信息，按优先级修复：
   P0 — 真正的类型错误（会导致运行时崩溃的）
   P1 — 类型推断不准确的（用 as 断言或补充类型注解）
   P2 — 未使用变量警告

3. 重点检查项：
   - 所有 API 调用函数的参数和返回值类型是否与后端 DTO 一致
   - 所有 useState/useReducer 的状态类型是否完整
   - dnd-kit 的事件处理函数类型是否正确
   - Formily 的 Schema 类型是否与后端 LabelSchema.schema 字段一致
   - 页面间路由跳转的 state 类型是否正确

4. 修复原则：
   - 禁止使用 any（项目规范）
   - 动态数据用 Record<string, unknown>
   - 真不确定的类型用 unknown + 类型守卫
   - 确需类型断言用 as unknown as TargetType 两步转换

5. 验证：
   - 修复所有报错直到 npx tsc --noEmit 零错误通过
   - 启动 dev server 确认页面正常渲染

请注意：类型检查零错误是项目的基本要求。
```
