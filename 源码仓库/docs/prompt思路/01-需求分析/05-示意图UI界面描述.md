---

## UI 设计方向分析

### 整体设计语言

| 维度 | 方向 |
|------|------|
| 设计风格 | 现代简约企业级，去 emoji 纯文本图标，Ant Design 6 组件体系 |
| 主色调 | 靛蓝 `#4f46e5`（品牌色），翠绿 `#10b981`（成功/通过），橙黄 `#f59e0b`（警告/审核中），红色 `#ef4444`（打回/错误） |
| 布局模式 | 左-主-右 三栏是核心模式（Schema设计器、标注工作台、审核中心），列表页用单栏表格 |
| 移动端 | ≤768px 自动切换：表格精简为2列+详情弹窗，三栏变单栏+抽屉切换 |
| 无刷新体验 | SPA 路由 + Zustand 全局状态 + Axios 401 自动续签 Token |

### 1. 登录/注册页

**布局**：左右分栏 — 左侧品牌展示区（渐变背景 + Logo + Slogan），右侧表单区（登录/注册切换）

**设计要点**：
- 左侧紫色渐变背景 `#4f46e5 → #7c3aed`，展示平台名称和一句话描述
- 右侧白色卡片式表单，胶囊 Tab 切换登录/注册
- 注册表单含角色下拉 `owner/labeler/reviewer`，默认 `labeler`
- 登录表单仅邮箱+密码，注册多一个用户名
- 已登录用户访问 `/login` 自动跳转 `/datasets`（工作台入口）

**移动端**：左右分栏变上下堆叠，左侧品牌区缩小为顶部 banner

### 2. 首页

**布局**：全屏纵向滚动 — Hero 首屏 → 功能介绍卡片 → 角色卡片 → 工作流展示 → Schema 演示 → CTA

**设计要点**：
- Hero 区全屏高度，负 margin 撑满（`margin: -28px -32px`），星空动画背景（15 颗 SVG 四角星，12-28px，半透明紫色）
- 功能介绍 4 张卡片 + 图标，IntersectionObserver 触发淡入动画
- 角色卡片 3 列（Owner/Labeler/Reviewer），每张卡描述职责+可访问页面
- Schema Designer 演示区：展示实际拖拽搭建的交互 Demo
- 底部 CTA：已登录跳工作台，未登录跳注册

**移动端**：Hero 区高度自适应，卡片 1 列排列，Schema 演示区隐藏

### 3. 数据集管理

**布局**：列表页（单栏表格）+ 详情页（左侧条目队列 + 右侧数据预览/编辑）

**页面拆分为 3 个子路由**：
- `/datasets` — DatasetList：搜索 + 表格（名称/格式/条目数/创建时间）+ 分页
- `/datasets/:id` — DatasetDetail：条目表格 + 内联编辑 + 删除
- `/datasets/:id/import` — DataImport：文件上传 + 预览 + 批量导入

**设计要点**：
- 表格支持搜索过滤（名称），每页 20 条
- 条目 JSON 数据用语法高亮组件渲染（JsonHighlight）
- 内联编辑：双击单元格进入编辑模式（InlineName 组件）
- CSV 文件前端解析（csvParser.ts），JSONL 按行 split

**移动端**：表格仅保留名称+操作两列，其余信息点击名称弹出详情 Modal 查看

### 4. Schema 设计器（核心模块）

**布局**：三栏 — 左侧物料面板（180px）+ 中间画布（flex:1）+ 右侧属性面板（180px）

**物料面板 (Palette)**：
- 10 种可拖拽组件：单行文本 / 多行文本 / 单选 / 多选 / 下拉选择 / 日期 / 富文本 / JSON 编辑器 / LLM 交互 / ShowItem
- 每个组件卡片可直接拖拽到画布或点击添加

**中间画布 (Canvas)**：
- 顶部工具栏：`[+添加Tab] [+添加分组] [预览] [导入JSON] [导出JSON]`
- Tab 栏：多个 Tab 页签，可拖拽排序（dnd-kit `horizontalListSortingStrategy`），双击重命名，hover 显示删除 X
- 未分配区域：有 Tab 时，未归属任何 Tab 的字段/容器显示在顶部红色虚线分隔区
- 分组容器 (GroupShell)：蓝色虚线框，可折叠，容器内字段 DragOver 高亮
- 字段卡片 (FieldCard)：显示类型图标+标题，支持拖拽、点击选中、复制、删除

**属性面板 (PropertyEditor)**：
- 选中字段时：编辑 key / title / 类型 / 必填 / 只读 / 默认值 / 占位文本 / 校验规则 / 字段联动（条件显示）/ 所属 Tab / 所属容器
- 选中容器时：编辑标题 / 包含的字段列表
- 字段在容器内时，"所属 Tab"自动显示"(继承自容器)"并禁用

**设计要点**：
- 全局状态用 `useReducer`（SchemaDesignerContext），所有操作通过 Action dispatch
- 数据模型三层分离：Tab（含 fieldIds + groupIds）→ Group（含 fieldIds）→ Field
- 搭建产物为 JSON Schema，可导入/导出/版本管理
- Designer 预览和 Labeler 工作台渲染共用 Formily 引擎

**移动端**：三栏变为单栏，Palette 和 PropertyEditor 用 Drawer 抽屉滑出

### 5. 任务管理

**布局**：任务列表（表格）+ 创建/编辑弹窗 + 导出页

**页面拆分**：
- `/tasks` — TaskList：表格列（标题/标签/状态/策略/进度/截止/操作）
- `/tasks/create` / `/tasks/:id/edit` — 创建/编辑表单 Modal
- `/tasks/:id/export` — ExportPage：格式选择 + 字段映射配置 + 导出历史

**设计要点**：
- 标签用固定 8 色调色板循环（`TAG_COLORS`），保证同一标签在不同页面颜色一致
- 进度条 `Progress` 组件 + 数字 `completed_items/quota`
- 状态用 Segmented 切换筛选（全部/草稿/发布/暂停/结束）
- 导出页：左侧格式配置（JSON/JSONL/CSV/Excel + 字段映射），右侧导出历史列表

**移动端**：表格精简，创建/编辑用全屏 Modal

### 6. 标注工作台

**布局**：三栏 — 左侧条目队列（280px）+ 中间标注表单（flex:1）+ 右侧统计面板（可收起）

**左侧条目队列 (ItemQueue)**：
- 每条显示序号 + 状态标记（pending ○ / labeled ● 绿 / skipped ◐ 黄）
- 当前选中高亮紫色边框
- 底部进度条 `labeled/total`

**中间标注表单**：
- Schema 驱动渲染：TabbedForm（有 Tab 的 Schema）→ SchemaForm（无 Tab）→ FallbackForm（无 Schema 降级）
- 字段联动：根据条件显示/隐藏字段
- 草稿自动保存（3s 防抖 → localStorage）+ 恢复提示
- LLM 辅助：字段旁按钮触发 LLM 建议写入
- 1 分钟定时刷新：先保存草稿，再静默拉取最新任务/条目状态

**右侧统计面板 (MetaPanel)**：
- 任务信息卡片（标题/截止/奖励）
- 标注统计卡片（待标注/已标注/已跳过/总计）

**底部操作栏 (BottomBar)**：提交 / 跳过 按钮，固定在页面底部

**设计要点**：
- 状态机驱动：提交 → Item.labeled + Result.submitted → AI 审核 → Reviewer 操作
- 驳回后重新打开：可查看驳回理由（`last_result.comment`），修改后重新提交（`round+1`）

**移动端**：三栏变单栏，左侧队列用 Drawer，右侧统计收起到顶部

### 7. 审核中心

**布局**：列表页 + 详情工作台（复审/终审双视角切换）

**页面拆分**：
- `/review` — ReviewList：待审核任务列表（表格 + 状态筛选）
- `/review/:taskId` — 审核工作台：左侧 AI 分组侧栏 + 右侧审核详情
- `/review/:taskId/final` — 终审视角

**左侧 AI 分组侧栏**：
- 三个 Tab：建议通过 / 建议打回 / 转人工，每项标注数量
- 每条显示：编号 / AI 综合评分 / 标注员 / AI 判断结论 / 审核轮次
- Checkbox 多选 + 批量通过/批量打回按钮
- 已选计数实时更新

**右侧审核详情 (ReviewDetail)**：
- 提交内容（原始数据 vs 标注结果对比视图）
- AI 评分面板（多维度条形图 + 综合评分）
- 审核操作：通过（终审直接入库）/ 驳回（必填驳回理由）/ 修订（直接修改标注数据）
- 审计日志时间线（状态变更历史）

**设计要点**：
- `reviewable` 包含 `rejected` 状态的结果，确保驳回数据可被查看和重新操作
- 复审通过 → `final_review`，终审通过 → `warehouse`
- 任一审核阶段可打回 → `rejected`，打回后 Labeler 修改重新提交

**移动端**：审核列表和详情分两个页面，AI 侧栏用 Drawer

### 8. AI 预审监控

**布局**：双栏 — 左侧 AI 审核任务队列 + 右侧审核详情

**左侧任务队列**：
- 顶部统计卡片（总数/通过/打回/待处理）
- 任务卡片列表：任务名 / Agent 名 / 审核进度 / 最近审核时间

**右侧审核详情（4 区）**：
- 元数据区：任务信息 / 提交者 / 审核时间 / 耗时
- 维度评分区：每个维度分数 + 条形图可视化 + 理由
- AI 评语区：LLM 审核总结
- 处理日志区：进入队列 / 调用 LLM / 判定结果 的时间线

**设计要点**：
- 从 `ai_reviews` 表读取完整审核记录（非 `result.ai_scores` 快照）
- 失败记录显示错误信息，支持「重跑」按钮重新投递 MQ
- 点击任务卡片切换右侧详情

### 9. AI Agent 配置

**布局**：单栏表格 + 创建/编辑弹窗

**表格列**：名称 / 邮箱 / LLM 模型 / 状态（空闲/忙碌/离线）/ 审核统计（总数+通过率）/ 评分维度 / 操作

**弹窗表单**：
- 基本信息：名称 / 邮箱 / 密码
- System Prompt 模板（TextArea，等宽字体，支持 `{task}` `{item}` `{result}` `{dimensions}` 变量）
- LLM 模型：灰置固定模型标识，不可编辑
- 评分维度：动态添加/删除行（标识 + 显示名 + 权重 0-1），底部实时显示权重合计

**设计要点**：
- 定时 10s 刷新 Agent 状态和统计数据
- 池状态栏：容量 / 执行中 / 排队中，执行中接近容量时红色预警
- 默认 Prompt 包含字段联动空值说明，避免误判

### 10. 数据看板

**布局**：指标卡片行 + ECharts 图表区

- 顶部 4 张统计卡片（总任务数/进行中/已完成/标注员数）
- 任务进度图表（柱状图或进度条）
- 数据集统计（饼图或环形图）

### 11. 教程页

**布局**：左侧可折叠侧边栏导航 + 右侧 Markdown 内容区

- 侧边栏：JSON 配置驱动（`toc.json`），7 章 × ~3 节，章节图标+折叠展开
- 内容区：`import.meta.glob` 动态加载 MD 文件，零依赖 Markdown 渲染器
- 支持标题(h1-h4)、粗体、斜体、行内代码、代码块、链接、图片、引用、有序/无序列表、表格

**移动端**：侧边栏变 Drawer 抽屉

## 核心交互模式总结

| 模式 | 应用页面 | 实现方式 |
|------|---------|---------|
| 拖拽排序 | Schema 设计器、Tab 栏 | @dnd-kit `useSortable` + `SortableContext` |
| 点击选中 | Schema 设计器 FieldCard、审核侧栏条目 | `onPointerDown/Up` 坐标差 < 4px 判定（避免和拖拽冲突） |
| 三栏布局 | Schema设计器、标注工作台、审核中心 | Flex 布局，左右定宽 + 中 flex:1 |
| 移动端适配 | 全局 | ≤768px 三栏变单栏 + Drawer，表格精简列 |
| 状态流转 | 工作台提交、审核通过/驳回 | validate_transition → transit → audit_log |
| 草稿保存 | 标注工作台 | 3s 防抖 + localStorage，恢复时弹窗提示 |
| 自动刷新 | 标注工作台（1min）、AI 配置页（10s） | setInterval 定时轮询 |
| Token 续签 | 全局 | Axios 响应拦截器 401 → refresh → 重放 |

## 视觉规范

| 属性 | 值 |
|------|-----|
| 品牌色 | `#4f46e5` 靛蓝紫 |
