# 功能需求梳理

## 我的提示词

```
你是一个产品经理，我正在设计一个 AI 数据标注平台 "LabelHub"。请根据以下背景帮我梳理完整的功能需求：

平台目标：
- 任务负责人可拖拽搭建标注页面模板
- 标注员在线领取/分发/提交任务
- 平台基于可配置评测标准提供 AI 自动预审
- 多角色人工审核流转保证数据质量
- 支持多格式数据导出对接下游训练

请按以下维度梳理功能需求：
1. 任务管理模块 — 任务负责人后台应该有哪些功能
2. 标注页面动态搭建 — 可视化模板搭建器需要哪些物料组件和配置能力
3. 标注员工作台 — 标注员的完整操作链路
4. AI 自动预审 Agent — Agent 需要具备哪些能力，如何配置
5. 多角色人工审核流转 — 审核工作流的状态节点和流转规则
6. 多格式数据导出 — 导出格式、字段映射、下载历史

每个功能点写清楚：功能名称、功能描述、优先级（P0/P1/P2）
```

---

## AI 的回答

### 1. 任务管理模块 (P0)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 任务 CRUD | 标题、描述（富文本）、标签、奖励规则、截止时间、配额 | P0 |
| 状态机 | 草稿 → 发布中 → 已暂停 → 已结束，含发布前抽屉确认 | P0 |
| 数据集绑定 | 从已有数据集中选择题目集合 | P0 |
| Schema 绑定 | 从 Schema 库选择标注模板，发布时冻结版本快照 | P0 |
| 分发策略 | 先到先得 / 指派 / 配额抢单（任选其一） | P1 |
| 进度监控 | 已完成/总题目数、通过率、标注员贡献统计 | P2 |

### 2. 标注页面动态搭建 — 核心难点 (P0)

**物料组件清单（10 种）：**

| 物料 | 说明 | 优先级 |
|------|------|--------|
| 单行文本 | 基础短文本采集 | P0 |
| 多行文本 | 长文本/段落采集 | P0 |
| 单选 | 枚举类标注 | P0 |
| 多选 | 多值枚举标注 | P0 |
| 标签选择 | 自由标签 + 预设标签 | P1 |
| 富文本编辑器 | 带格式长文本 | P1 |
| 文件/图片上传 | 多媒体素材 | P2 |
| JSON 编辑器 | 结构化数据输入 | P1 |
| LLM 交互组件 | 字段级模型调用，输出可预填 | P1 |
| 展示项 (ShowItem) | 只读渲染题目原始数据 | P0 |

**搭建器能力：**

| 能力 | 说明 | 优先级 |
|------|------|--------|
| 左侧物料面板 + 中间画布 + 右侧属性面板 | 标准三栏拖拽布局 | P0 |
| 分组容器 (Group) | 字段分组，支持折叠 | P0 |
| 多 Tab 布局 | 标签页切换 | P1 |
| 字段属性配置 | 标题/key/类型/必填/默认值/占位提示 | P0 |
| 字段联动 | 条件显示 / 联动校验 | P1 |
| 自定义校验 | 必填/长度/正则/自定义函数 | P1 |
| Schema 导入/导出 | 搭建产物 JSON 可序列化可迁移 | P0 |

### 3. 标注员工作台 (P0)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 任务广场 | 搜索/筛选/任务卡片（标题/描述/题数/奖励/配额） | P0 |
| 领取任务 | 一键领取，配额管理 | P0 |
| Schema 驱动表单渲染 | 根据任务绑定的 Schema 动态渲染标注表单 | P0 |
| 上下题导航 | 上一题/下一题/快速跳题 | P0 |
| 草稿自动保存 | 字段变更自动保存，刷新恢复 | P1 |
| 提交校验 | 提交前校验表单完整性，错误提示 | P0 |
| LLM 辅助 | 字段级 AI 建议调用 | P1 |
| 打回处理 | 查看打回原因 + 审核意见 + 修改重提 | P0 |
| 我的数据 | 已提交/通过/打回的统计与列表 | P1 |

### 4. AI 自动预审 Agent (P1)

| 能力 | 描述 | 优先级 |
|------|------|--------|
| 可配置 Prompt 模板 | 任务负责人在后台配置系统提示词 + 评分标准 | P1 |
| 多维度评分 | 相关性/准确性/格式合规/安全性…可自定义 | P1 |
| 结构化输出 | Function Calling / JSON Mode，避免裸文本解析 | P1 |
| 结论分类 | 通过/打回/人工复核 | P0 |
| 审核结果记录 | 评分详情 + 原始 Prompt 一并入库可追溯 | P0 |
| 异步队列 | 提交后自动入队，不阻塞标注操作 | P0 |
| 失败重试 | LLM 调用失败后支持手动重跑 | P1 |
| Agent 管理 | 创建/编辑/删除/状态监控 | P1 |

### 5. 审核流转 (P0)

审核工作流 6 态：

```
submitted → ai_reviewing → review → final_review → warehouse
               ↓              ↓          ↓
            rejected ← ─ ─ ─ ─ ┘          │
               ↑__________________________┘
```

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 审核列表 | 按任务/状态筛选，显示待审核/已审核 | P0 |
| 审核详情 | 原始数据 + 标注结果 + AI 评分 + 审核操作 | P0 |
| 通过/打回 | 打回必须附理由 | P0 |
| 批量审核 | 多选批量通过/打回 | P1 |
| 审计时间线 | 完整的状态变更历史 | P1 |

### 6. 导出 (P1)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 格式支持 | JSON / JSONL / CSV / Excel | P1 |
| 异步导出 | 后台生成，不阻塞操作 | P1 |
| 字段映射 | 选择导出字段 + 重命名 | P2 |
| 导出历史 | 历史记录 + 下载链接 | P2 |
