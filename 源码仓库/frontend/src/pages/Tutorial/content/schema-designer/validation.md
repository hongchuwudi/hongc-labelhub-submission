# 校验规则

Schema Designer 支持为每个字段配置多条校验规则，标注员提交时前端和后端双重校验。

## 支持的规则类型

| 规则 | 类型值 | 参数 | 说明 |
|------|------|------|------|
| 必填 | required | 无 | 字段不能为空 |
| 最小长度 | minLength | 数值 | 字符串最少字符数 |
| 最大长度 | maxLength | 数值 | 字符串最多字符数 |
| 正则匹配 | pattern | 正则表达式 | 格式校验（如手机号、身份证） |
| 最小值 | min | 数值 | 数字类型最小值 |
| 最大值 | max | 数值 | 数字类型最大值 |

## 配置方式

![Schema Validation](https://hc-base.oss-cn-beijing.aliyuncs.com/tutorial/schema-validation.png)


在右侧属性面板的「校验规则」区域：
1. 点击预设规则快速添加（必填、最小长度、最大长度）
2. 或手动输入正则表达式 + 错误提示文案
3. 每条规则可编辑错误提示信息
4. 支持删除已添加的规则

## 校验规则数据结构

```json
{
  "type": "minLength",
  "value": 2,
  "message": "至少输入 2 个字符"
}
```

## 校验时机

- **前端校验**：标注员点击提交时，逐字段执行所有规则，不通过则高亮对应字段并显示错误信息
- **预览模拟**：Schema Designer 预览模式下提供「校验」按钮，可模拟提交校验
- **规则可组合**：一个字段可同时配置多条规则（如必填 + 最大长度 + 正则）

> required 规则可通过两种方式设置 —— 勾选字段的「必填」属性，或在规则列表中添加 required 规则，两者效果相同。
